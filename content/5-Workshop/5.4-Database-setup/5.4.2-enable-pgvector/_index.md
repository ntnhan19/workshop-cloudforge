To power our AI semantic search capabilities, we need our PostgreSQL database to understand and store vector embeddings. Amazon RDS for PostgreSQL supports the `pgvector` extension natively.

#### 1. Connect to the Database
Since our database is securely placed in a Private Subnet (following the Zero-Trust model), it cannot be accessed directly from the public internet. To connect, you will need to use a client (such as `psql`, pgAdmin, or DBeaver) from within the VPC. 

*Common approaches for this workshop:*
- Use an **EC2 Bastion Host** or **AWS Cloud9** environment deployed in the Public Subnet.
- Use **AWS Systems Manager (SSM) Session Manager** port forwarding to connect securely from your local machine.

Using the credentials you created in the previous step:
- **Host:** *[Your RDS Endpoint URL]*
- **Port:** 5432
- **Database:** `cloudforge_db`
- **Username:** `postgres`
- **Password:** *[Your Master Password]*

#### 2. Enable the Extension
Once connected to the `cloudforge_db` database, execute the following SQL command to enable the vector extension:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

#### 3. Verify the Installation
To ensure the extension is installed and working correctly, you can run:

```sql
SELECT * FROM pg_extension WHERE extname = 'vector';
```

If the query returns a row, `pgvector` is successfully activated! Our database is now ready to store high-dimensional vectors generated by Amazon Titan.
